# 테스트 계획 (Timetable)

## 1) 테스트 목표

핵심 사용자 플로우와 회귀 포인트를 아래 기준으로 검증한다.

- **핵심 사용자 플로우**
  - 사용자가 시간표 화면을 열고(초기 렌더링), 셀을 선택한 뒤, 과목(항목)을 추가/수정/삭제할 수 있어야 한다.
  - 페이지 새로고침 이후에도 localStorage에 저장된 시간표가 복원되어야 한다.
- **회귀 포인트**
  - 셀 선택 상태(선택/비선택 시각 상태)가 의도대로 유지되는지
  - CRUD(추가/수정/삭제) 이후 즉시 UI와 저장소가 일치하는지
  - 저장 데이터가 있을 때/없을 때 초기 상태가 올바른지

---

## 2) 테스트 시나리오 (최소 6개)

아래 시나리오는 Given-When-Then 형식으로 정의한다.

### 시나리오 1. 초기 렌더링

- **Given** 사용자가 시간표 페이지에 처음 접속했고, localStorage에 시간표 데이터가 없다.
- **When** 페이지가 렌더링된다.
- **Then** 기본 시간표 그리드가 표시되고, 사전에 추가된 항목 없이 빈 상태로 보인다.

### 시나리오 2. 셀 선택

- **Given** 시간표가 렌더링되어 있고 선택되지 않은 셀이 있다.
- **When** 사용자가 특정 셀을 클릭한다.
- **Then** 해당 셀이 선택 상태로 변경되고(시각적 하이라이트), 현재 선택된 슬롯 정보가 액션(추가/수정) 대상이 된다.

### 시나리오 3. 항목 추가

- **Given** 사용자가 특정 셀을 선택했고, 추가 폼(또는 다이얼로그)에 접근 가능하다.
- **When** 과목명/장소/교수명 등 필수 입력값을 입력하고 저장한다.
- **Then** 선택한 셀(또는 해당 시간 슬롯)에 새 항목이 렌더링되고 localStorage에도 반영된다.

### 시나리오 4. 항목 수정

- **Given** 기존에 추가된 항목이 화면에 보이고 수정 액션이 가능하다.
- **When** 사용자가 항목 정보를 변경한 뒤 저장한다.
- **Then** 변경된 값이 즉시 UI에 반영되고 localStorage 데이터도 최신 값으로 갱신된다.

### 시나리오 5. 항목 삭제

- **Given** 기존에 추가된 항목이 화면에 보이고 삭제 액션이 가능하다.
- **When** 사용자가 해당 항목을 삭제한다.
- **Then** 항목이 UI에서 사라지고 localStorage에서도 제거된다.

### 시나리오 6. localStorage 복원

- **Given** localStorage에 유효한 시간표 데이터가 저장되어 있다.
- **When** 사용자가 페이지를 새로고침하거나 재접속한다.
- **Then** 저장된 시간표가 자동으로 복원되어 동일한 상태로 렌더링된다.

## 3) Playwright MCP 실행 가이드

### 서버 실행

- 개발 서버 포트 예시: **3000**
- 실행 명령 예시:

```bash
npm run dev -- -p 3000
```

### MCP 브라우저 도구 실행 시 `ports_to_forward`

- Playwright MCP 호출 시 아래처럼 포트를 전달한다.
- 예시: `ports_to_forward: [3000]`

### 주요 셀렉터 전략

- **권장:** 안정적인 E2E를 위해 핵심 요소에 `data-testid`를 부여한다.
  - 예: `data-testid="timetable-cell-mon-1"`, `data-testid="add-item-button"`, `data-testid="save-item-button"`
- 텍스트 기반 셀렉터(`getByText`)는 i18n/문구 변경에 취약하므로 최소화한다.
- 구조 기반 셀렉터(CSS 깊은 경로)는 DOM 변경에 취약하므로 지양한다.

## 4) 스크린샷 캡처 체크포인트 (시각 변경 검증)

시간표 셀의 상태 변화(특히 선택/해제, 항목 표시/미표시)는 시각 회귀 가능성이 높으므로 아래 체크포인트에서 캡처한다.

1. **셀 선택 직후**: 선택 하이라이트가 적용된 상태
2. **항목 추가 직후**: 선택한 셀에 항목이 표시된 상태
3. **항목 수정 직후**: 변경된 텍스트가 셀에 반영된 상태
4. **항목 삭제 직후**: 항목이 제거된 빈 셀 상태
5. **새로고침 후 복원 상태**: localStorage 데이터가 복원된 화면

> 권장 파일명 예시: `artifacts/timetable-selected-cell.png`, `artifacts/timetable-after-add.png`
